这是一组用来构建并处理 Go 源码的程序套件。 本套件中的程序并不直接运行，而是通过 go 程序调用。

运行这些程序最常见的方式就是作为 go 程序的子命令，例如 go fmt。 这样运行时，该命令会对完整的 Go 源代码包进行操作， go 程序通过适当的实参来调用基本的二进制程序以进行包级处理。

这些程序也可以作为独立的二进制程序，结合未修改的参数， 使用 go tool 的子命令来运行，比如 go tool cgo。 这里的绝大多数命令主要用于调试。 有些命令，比如 pprof，只能通过 go tool 子命令来操作。

最后 fmt 和 godoc 命令被安装为常规二进制文件， 成为 gofmt 和 godoc，因为它们经常被引用。

点击下方链接可获得更多文档、调用方法和使用细节。

名称      简介
go      go 程序管理Go源代码以及运行这里列出的其他命令。详细使用方法请参见命令文档。
cgo      Cgo 能够创建调用C代码的Go包。
cover      Cover 是一个用于创建和分析由 "go test -coverprofile"生成的覆盖评估的程序。
fix      Fix 用于发现使用语言和库的旧功能的 Go 程序，并将其改写为使用新的功能。
fmt      Fmt 格式化 Go 包，它也可以作为一个独立的 gofmt 命令使用，有更多的通用选项。
godoc      Godoc 提取并生成 Go 包的文档。
vet      Vet 检查 Go 源代码并报告可疑的结构，例如 Printf 调用的参数与格式字符串不一致。

这是一个简略的列表。关于编译器的文档和更多信息，请参见完整的命令参考。
